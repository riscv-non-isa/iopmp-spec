[#IOPMP_SE]
=== Source-enforcement 
When some requestors already implement firewalls with functionality similar to the IOPMP, users can configure the IOPMP to apply rule checking only to unprotected requestors. This enables IOPMP simplification to support either a single requestor or multiple requestors with identical access permissionsâ€”a common configuration in embedded systems. For example, <<IOPMP_SE_Example>> illustrates a scenario where RISC-V harts implement PMP as the firewall, and IOPMP-SE is applied only to heterogeneous cores and general DMA.

Source-enforcement (IOPMP-SE) is a simplified IOPMP configuration that assumes a single requestor or multiple requestors with identical permissions. This implementation requires only one SRCMD Table entry and allows the RRID field to be ignored when all transactions originate from requestors with identical permissions.
